# Lab 4: VLAN (802.1q) — Тегирование трафика

В этой лабораторной работе мы разберемся, как работает разделение физической сети на изолированные логические сети с помощью VLAN (Virtual Local Area Network), а также как настроить Inter-VLAN маршрутизацию ("Router-on-a-stick").

## Запуск лабы
Для автоматического развертывания стенда выполните:
```bash
cd /root/linux_network_labs/lab4_vlan
bash setup.sh
```

## Что происходит под капотом?

1. Создаются три Network Namespaces: `host1`, `host2` и `router`.
2. Создается виртуальный мост (коммутатор) `br0`, к которому подключены все узлы через veth-интерфейсы. Все они физически находятся на одном L2-сегменте (один кабель).
3. На `host1` настраивается VLAN-интерфейс (`veth-h1.10`) с тегом **10** (IP: 10.0.10.10).
4. На `host2` настраивается VLAN-интерфейс (`veth-h2.20`) с тегом **20** (IP: 10.0.20.20).
5. На `router` настраиваются два VLAN-интерфейса: `10` (IP: 10.0.10.1) и `20` (IP: 10.0.20.1) и включается IP-форвардинг (`net.ipv4.ip_forward=1`).

## Демонстрация
Без роутера, если бы `host1` попытался пропинговать `host2`, пакеты бы не дошли. Несмотря на то, что они воткнуты в один свитч, теги VLAN у них разные: `10` и `20`. ОС и свитч не пропускают трафик чужого VLAN.

Однако, после настройки роутера, пакет из `host1` доходит до роутера по VLAN 10. Роутер видит, что целевой IP находится в подсети VLAN 20, перекладывает пакет в `veth-r.20` (меняя тег с 10 на 20), и отправляет обратно в свитч. `host2` получает его по VLAN 20!

Убедитесь, что пинг работает через роутер:
```bash
ip netns exec host1 ping -c 4 10.0.20.20
```

Проследить путь пакета можно запустив `tcpdump` на мосту `br0`:
```bash
tcpdump -i br0 -e -n vlan
```
Вы увидите, как туда ходят пакеты сначала с `vlan 10`, а потом с `vlan 20`!
