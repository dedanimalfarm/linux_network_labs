# Lab 2: L4/L7 Load Balancing with HAProxy

В этой лабораторной работе мы настроим балансировщик нагрузки HAProxy, который будет распределять HTTP-трафик между тремя простыми веб-серверами по алгоритму Round Robin.

## Запуск лабы
Для автоматического развертывания стенда выполните:
```bash
cd /root/linux_network_labs/lab2_loadbalancer
bash setup.sh
```

## Что происходит под капотом?

1. Скрипт поднимает три простых веб-сервера на Python на портах `8081`, `8082`, `8083`.
2. Запускается HAProxy с конфигурацией из файла `haproxy.cfg`.
3. HAProxy слушает порт `8080` и проксирует запросы на бэкенды.

## Проверка балансировки
Отправьте несколько запросов к нашему балансировщику:
```bash
curl http://localhost:8080
curl http://localhost:8080
curl http://localhost:8080
```
Вы должны увидеть ответы от разных серверов (`Hello from Server 1`, `Server 2`, `Server 3`), что подтверждает работу алгоритма **Round Robin**.

## Проверка Healthchecks (отказоустойчивости)
Давайте проверим, как балансировщик справится с падением одного из серверов. Убьем первый сервер:
```bash
pkill -f "http.server 8081"
```
Повторите запросы к балансировщику:
```bash
curl http://localhost:8080
```
Теперь HAProxy поймет, что первый сервер недоступен (благодаря параметру `option httpchk GET /` в конфигурации), и будет перенаправлять трафик только на Server 2 и Server 3.
