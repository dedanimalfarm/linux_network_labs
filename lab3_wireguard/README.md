# Lab 3: VPN Tunnels with WireGuard

В этой лабораторной работе мы настроим защищенный VPN-туннель между двумя изолированными узлами сети (используя Linux Network Namespaces для эмуляции) с помощью WireGuard.

## Запуск лабы
Для автоматического развертывания стенда выполните:
```bash
cd /root/linux_network_labs/lab3_wireguard
bash setup.sh
```

## Что происходит под капотом?

1. Создаются два сетевых пространства имен `ns1` и `ns2`.
2. Между ними поднимается "физический" линк (veth-пары) в подсети `10.0.0.0/24`. Узлы `10.0.0.1` и `10.0.0.2` могут общаться друг с другом без шифрования.
3. Генерируются приватные и публичные ключи WireGuard: `wg genkey` и `wg pubkey`.
4. В обоих неймспейсах создаются интерфейсы `wg0`, настраиваются ключи, IP-адреса туннеля (`192.168.100.1` и `192.168.100.2`) и пиры (Peers).
5. Весь трафик к подсети `192.168.100.0/24` теперь идет через зашифрованный туннель на порту `51820`.

## Проверка связи
Проверьте туннель, пустив пинг через виртуальную частную сеть из `ns1` в `ns2`:
```bash
ip netns exec ns1 ping -c 4 192.168.100.2
```

Также вы можете посмотреть статус интерфейса WireGuard:
```bash
ip netns exec ns1 wg show
```
В выводе вы увидите информацию о пире, публичном ключе, используемом endpoint (`10.0.0.2:51820`) и статистику переданных данных, что доказывает, что туннель поднят.
